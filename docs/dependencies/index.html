/lib/container-common

Libraries in /lib/container-common will be installed into the servlet container such that the container and the contained web applications share them.

You can add your database driver .jar to this directory and initportal will install your database driver .jar into the servlet container for you. This can be a nice convenience in development.

/lib/provided

This directory contains .jars required to compile uPortal but which are expected to be provided at runtime by the application container and so are not deployed with uPortal.

.jar name description acquired from license

/lib/runtime

These libraries are not included at compile time but are deployed with uPortal at runtime. Currently this directory is empty. Were log4j not required by the deprecated legacy LogService uPortal class, it would go here.

/lib/test

Libraries required to compile and/or run the unit tests for the project, but not required to be available at compile time or runtime for the application.
